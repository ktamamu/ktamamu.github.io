Route53とオンプレDNSにおける条件付きフォワーダーと委任の構成比較

従来の構成：条件付きフォワーダー
リゾルバーDNS設定:
zone "intra.example.com" {
  forwarders { 10.1.1.2; };
}
zone "delegate-test.intra.example.com" {
  forwarders { 10.128.130.26; };
}
1
クライアントが hoge.delegate-test.intra.example.com の名前解決を要求
2
リゾルバーDNSサーバーが条件をチェック：
delegate-test.intra.example.com に該当 → Route 53エンドポイントに転送
3
Route 53インバウンドエンドポイントがプライベートホストゾーンから回答を取得
4
結果をクライアントに返却
新しい構成：DNS委任
リゾルバーDNS設定:
zone "intra.example.com" {
  forwarders { 10.1.1.2; };
}
// サブドメインの設定は不要！

権威DNS委任設定:
delegate-test IN NS ns1.delegate-test
ns1.delegate-test IN A 10.128.130.26
// 正式な委任関係
1
クライアントが hoge.delegate-test.intra.example.com の名前解決を要求
2
リゾルバーDNSサーバーが権威DNSサーバーに問い合わせ
3
権威DNSサーバーが委任情報を返却：
「delegate-test.intra.example.com は 10.128.130.26 に委任されています」
4
リゾルバーDNSサーバーが委任先のRoute 53エンドポイントに問い合わせ
5
結果をクライアントに返却
従来の方法：条件付きフォワーダー
  • 設定場所：リゾルバーDNSサーバー
  • 管理方法：ドメインごとに転送ルールを設定
  • DNS階層：非標準的な構成
  • メンテナンス：サブドメイン追加時に設定変更が必要
  • 統制：リゾルバー側で制御
新しい方法：DNS委任
  • 設定場所：権威DNSサーバー
  • 管理方法：標準的なDNS委任
  • DNS階層：標準的な階層構造
  • メンテナンス：権威サーバーで一元管理
  • 統制：権威サーバー側で制御
