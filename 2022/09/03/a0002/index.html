# IPv6用のインスタンス作成

安くて素敵。

WebARENA Indigo™ IPv6対応プランの提供開始 (opens new window)

FWが設定できないからOSで設定する必要がある。

# Google Domainsに登録する

初めてAAAAレコードで登録した。

# Macから接続設定

ネットワーク設定でIPv6のところを自動にしておく。 V6プラスを使ってるおうちなので、ISP的には問題なく接続できた。（SSH）

こんなコマンド初めて打った。楽しい。

$ ping6 [IPv6 Adrerss]
$ dig [Domain Name] AAAA

SSHは特にAddressFamily inet6とかはしなくても接続できそう

# 既存のIndigo上のインスタンスとの接続

元から使ってたIndigo上のUbuntuインスタンスはIPv6のアドレスがなく、今回のサーバと連携できなかった。

でも、このインスタンスにスナップショットとかで移行して、

WebARENA Indigo™ 「IPv4/IPv6デュアルスタック対応プラン」「インスタンス複製機能」の提供開始、対応OSテンプレート追加について (opens new window)

この設定すれば多分使えると思ってる（やってない）

IPv6を有効化する（Ubuntu系） (opens new window)

# nginxとcertbot入れてみる

nginx入れてcertbot実行したらエラーが出た。

An unexpected error occurred:
requests.exceptions.ConnectionError: HTTPSConnectionPool(host='supporters.eff.org', port=443): Max retries exceeded with url: /subscribe/certbot (Caused by NewConnectionError('<urllib3.connection.HTTPSConnection object at 0x7fb227f1e620>: Failed to establish a new connection: [Errno 101] Network is unreachable'))

まぁでも、その上の行に成功って出てるからいいっしょ、ってことでひとまず放置

Successfully deployed certificate for [Domain Name] to /etc/nginx/sites-enabled/default
Congratulations! You have successfully enabled HTTPS on [Domain Name]

Macからは問題なく接続できた。
